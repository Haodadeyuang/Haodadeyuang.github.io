CAS实现无锁线程安全

CAS

Compare And Set ， 比较后进行赋值
若要将 A 赋值为 B
1、计算出赋值后A的预期值 expect
2、获得 A 在内存中的最新值 prev（JVM中用volatile关键字修饰）
3、CAS，比较 prev跟 A 的值，若一致，则将expect赋值给A，返回true，若不一致，从步骤1开始自旋。

一般情况下，CAS方法比加锁的效率高很多，因为加锁解锁需要在用户态跟内核态之间切换，CAS是乐观锁，默认自旋花费较低。

CAS方法在Java中的实现调用了native方法，其底层实现是将CAS操作在汇编时转换为原语，即若干机器指令组成的不可被中断的操作。

ABA问题

CAS存在快慢线程问题
快线程启动 读到P的值 A ，慢线程启动 读到P的值 A 。

快线程将其P的值修改为 B，再次读取到P的值 B，将其修改为A

此时慢线程使用CAS方法，比较 P的最新值就是预期的 A，虽然也修改成功了，但是无法确定P在这期间有没有变化。

假设有一个这样的链表 head-> A -> B -> C

快、慢线程读到 head的next指向 A

而后快线程移除A，移除B，放入A，链表变为head-> A -> C

慢线程调用CAS方法，比较head->next仍为A，可以 修改，但是此时链表已经发生了变化

抽象类、接口

抽象类

抽象类生来就是来被继承的，它可以只定义某个方法的签名，具体实现由继承它的子类去完成。

使用抽象类可以规范子类的写法，因为子类在继承抽象类的时候，必须将所有的未实现抽象方法都重写。

抽象类本身无法被实例化，但是其引用的对象可以由子类强制向上转型，我们在使用父类所创造出来的引用时，可以不关心这个引用内部究竟是哪个子类，只需要使用其方法就行了。

如果子类没有完全重写父类的所有抽象方法，则其也是抽象类，也就是说，只要有抽象方法的存在，这个类就是抽象类。

接口

接口是更抽象的类，它要求接口里的方法都只能是抽象的，同时没有任何成员变量。

equals方法与==

对于基本数据类型，==比较的总是两个值是否一样，而对于对象，==比较的是两个对象在内存中的地址是否一样，即它们是否为同一个对象。

对于含有方法的对象中的equlas方法在没有进行重写*的情况下，与*==完全一致，即比较的是两个对象的地址。

不过比较常用的String对象的equals方法就被进行了重写，用于判断两个String对象的内容是否一样。

2、在Integer跟int进行==比较时，Integer会自动拆箱，最终比较的还是两个int之间的值，与对象无关。

equals方法与hashCode

hashcode也是可以被重写的，所以实际上它与equals方法没有必然的关系，但是这样子就失去了它存在的意义了，所以我们人为规定了它的使用场景。

在默认情况下，它返回的是这个对象根据一个散列函数计算出来的散列值。对于同一个对象，不管进行多少次hashCode方法，返回值都应该是一致的。根据散列函数的特性，散列值一样的对象不一定是同一个对象，而同一个对象的散列值一定相等。

这个散列值在两个对象进行比较时可以更快的帮助我们锁定其位置，对于JAVA中的集合set，它是不允许存在两个一样的值的，我们每次添加新元素进去的时候，都要检查这个元素是否已经出现过了，假如这个集合中已经存在了非常多的元素，就只能一个个去使用equals方法进行比较，效率十分低下，而hashCode就像是把这个集合分为一个个桶，每个元素都有一个桶号，在我们对两个元素进行equlas方法比较时，它会先计算这个两个元素的hashCode，假如不一样，就直接插入到对应的位置，假如它们的hashCode一样，再进行equals方法，这样子就能大大减少equals方法的调用次数。

结论

1、如果两个对象equals，Java运行时环境会认为他们的hashcode一定相等。
2、如果两个对象不equals，他们的hashcode有可能相等。
3、如果两个对象hashcode相等，他们不一定equals。
4、如果两个对象hashcode不相等，他们一定不equals。

关于这两个方法的重要规范：

规范1：若重写equals(Object obj)方法，有必要重写hashcode()方法，确保通过equals(Object obj)方法判断结果为true的两个对象具备相等的hashcode()返回值。说得简单点就是：“如果两个对象相同，那么他们的hashcode应该相等”。不过请注意：这个只是规范，如果你非要写一个类让equals(Object obj)返回true而hashcode()返回两个不相等的值，编译和运行都是不会报错的。不过这样违反了Java规范，程序也就埋下了BUG。

规范2：如果equals(Object obj)返回false，即两个对象“不相同”，并不要求对这两个对象调用hashcode()方法得到两个不相同的数。说的简单点就是：“如果两个对象不相同，他们的hashcode可能相同”。

前言

刚转Java，为面试作下准备

Java基础

1.JDK 和 JRE 有什么区别？

JDK是开发包，包含了JRE，JRE又包含了JVM

JRE是运行环境，JVM是JAVA虚拟机。

2.== 和 equals 的区别是什么？

==指的是内存中地址是否相同，而equals指的是内存中内容是否相同。

new方法会重新申请一个对象，而c跟d显然不是同一个对象，在内存中的地址不同。

String直接申明时(a,b)，不会在堆中申请，而是扣留在常量池里，所以后续申请时，会直接指向同一块引用。

3.两个对象的 hashCode()相同，则 equals()也一定为 true，对吗？

错了，两个对象的equals()相同，则它们的hashCode()一定相同，而相同的哈希值可能是由不同的字符串通过同一个算法得来的。

4.final 在 java 中有什么作用？

final 最后的，最终的。

它可以修饰对象、方法、变量

修饰对象时，该对象无法再被继承

修饰方法时，该方法无法再被重写

修饰变量时，该变量无法再被修改，如果指向的是一个引用变量，那引用的对象不可变，但是引用的对象的值是随意变化的，同时，如果是final修饰了类的成员变量，那必须在申明时候就赋初始值。

5.java 中的 Math.round(-1.5) 等于多少？

-1。

Math.round()方法得到最接近参数的整数，将参数＋0.5后向下取整。

6.String 属于基础的数据类型吗？

不属于，它是引用类型，是类。

八个基本数据类型。

  • 整数类型：byte，short，int，long （1字节、2字节、4字节、8字节）
  • 浮点数类型：float，double （4字节、8字节）
  • 字符类型：char （2字节）
  • 布尔类型：boolean （JVM中，单个布尔值时以int为样，即4个字节，布尔数组则每个占一个字节）

7.java 中操作字符串都有哪些类？它们之间有什么区别？

String、StringBuffer、StringBuilder

String : final修饰，String类的方法都是返回new String。即对String对象的任何改变都不影响到原对象，对字符串的修改操作都会生成新的对象。
StringBuffer : 对字符串的操作的方法都加了synchronized，保证线程安全。
StringBuilder : 不保证线程安全，在方法体内需要进行字符串的修改操作，可以new StringBuilder对象，调用StringBuilder对象的append、replace、delete等方法修改字符串。

不一样，前者在常量池中，后者在堆中。

移除元素

给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素，并返回移除后数组的新长度。

不要使用额外的数组空间，你必须仅使用 O(1) 额外空间并 原地 修改输入数组。

元素的顺序可以改变。你不需要考虑数组中超出新长度后面的元素。

示例 1:

给定 nums = [3,2,2,3], val = 3,

函数应该返回新的长度 2, 并且 nums 中的前两个元素均为 2。

你不需要考虑数组中超出新长度后面的元素。
示例 2:

给定 nums = [0,1,2,2,3,0,4,2], val = 2,

函数应该返回新的长度 5, 并且 nums 中的前五个元素为 0, 1, 3, 0, 4。

注意这五个元素可为任意顺序。

你不需要考虑数组中超出新长度后面的元素。

说明:

为什么返回数值是整数，但输出的答案是数组呢?

请注意，输入数组是以「引用」方式传递的，这意味着在函数里修改输入数组对于调用者是可见的。

你可以想象内部操作如下:

// nums 是以“引用”方式传递的。也就是说，不对实参作任何拷贝
int len = removeElement(nums, val);

// 在函数里修改输入数组对于调用者是可见的。
// 根据你的函数返回的长度, 它会打印出数组中 该长度范围内 的所有元素。
for (int i = 0; i < len; i++) {
print(nums[i]);
}
