Java基础

1.JDK 和 JRE 有什么区别？

JDK是开发包，包含了JRE，JRE又包含了JVM

JRE是运行环境，JVM是JAVA虚拟机。

2.== 和 equals 的区别是什么？

==指的是内存中地址是否相同，而equals指的是内存中内容是否相同。

new方法会重新申请一个对象，而c跟d显然不是同一个对象，在内存中的地址不同。

String直接申明时(a,b)，不会在堆中申请，而是扣留在常量池里，所以后续申请时，会直接指向同一块引用。

3.两个对象的 hashCode()相同，则 equals()也一定为 true，对吗？

错了，两个对象的equals()相同，则它们的hashCode()一定相同，而相同的哈希值可能是由不同的字符串通过同一个算法得来的。

4.final 在 java 中有什么作用？

final 最后的，最终的。

它可以修饰对象、方法、变量

修饰对象时，该对象无法再被继承

修饰方法时，该方法无法再被重写

修饰变量时，该变量无法再被修改，如果指向的是一个引用变量，那引用的对象不可变，但是引用的对象的值是随意变化的，同时，如果是final修饰了类的成员变量，那必须在申明时候就赋初始值。

5.java 中的 Math.round(-1.5) 等于多少？

-1。

Math.round()方法得到最接近参数的整数，将参数＋0.5后向下取整。

6.String 属于基础的数据类型吗？

不属于，它是引用类型，是类。

八个基本数据类型。

  • 整数类型：byte，short，int，long （1字节、2字节、4字节、8字节）
  • 浮点数类型：float，double （4字节、8字节）
  • 字符类型：char （2字节）
  • 布尔类型：boolean （JVM中，单个布尔值时以int为样，即4个字节，布尔数组则每个占一个字节）

7.java 中操作字符串都有哪些类？它们之间有什么区别？

String、StringBuffer、StringBuilder

String : final修饰，String类的方法都是返回new String。即对String对象的任何改变都不影响到原对象，对字符串的修改操作都会生成新的对象。
StringBuffer : 对字符串的操作的方法都加了synchronized，保证线程安全。
StringBuilder : 不保证线程安全，在方法体内需要进行字符串的修改操作，可以new StringBuilder对象，调用StringBuilder对象的append、replace、delete等方法修改字符串。

8.String str=”i”与 String str=new String(“i”)一样吗？

不一样，前者在常量池中，后者在堆中。

11.抽象类必须要有抽象方法吗？

  1. 抽象类可以不含有抽象方法
  2. 如果一个类包含抽象方法，则该类必须是抽象类

12.普通类和抽象类有哪些区别？

  • 抽象类不能被实例化
  • 抽象类可以有抽象方法，抽象方法只需申明，无需实现
  • 含有抽象方法的类必须申明为抽象类
  • 抽象类的子类必须实现抽象类中所有抽象方法，否则这个子类也是抽象类
  • 抽象方法不能被声明为静态
  • 抽象方法不能用 private 修饰
  • 抽象方法不能用 final 修饰

13.抽象类能使用 final 修饰吗？

不能，因为它必须被继承实现。

14.接口和抽象类有什么区别？

抽象类可以实现方法，而接口只能包含方法签名。

接口是更抽象的抽象类，不含任何字段。

接口可以有default方法，可以被重写，但并不是必须的。

七、异常

74.throw 和 throws 的区别？

throw是“动作”，而throws是“声明”，表示本方法可能抛出哪些异常

1、throws出现在方法函数头；而throw出现在函数体。
2、throws表示出现异常的一种可能性，并不一定会发生这些异常；throw则是抛出了异常，执行throw则一定抛出了某种异常。
3、两者都是消极处理异常的方式（这里的消极并不是说这种方式不好），只是抛出或者可能抛出异常，但是不会由函数去处理异常，真正的处理异常由函数的上层调用处理。
