快乐数

编写一个算法来判断一个数 n 是不是快乐数。

「快乐数」定义为：对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和，然后重复这个过程直到这个数变为 1，也可能是 无限循环 但始终变不到 1。如果 可以变为 1，那么这个数就是快乐数。

如果 n 是快乐数就返回 True ；不是，则返回 False 。

示例：

解法 一：使用集合

思路：只要在map中出现了，说明出现了一段循环，且这之间没有1，不会停止。

前提：在这道题中不可能出现无限不循环的情况，因为int类型最大值为为‭‭2 147 483 647‬‬， 平方和最大的数是1 999 999 999，平方和为1 + 81*9 = 724。任何数的平方和都在1到724之间，724次循环之内一定有重复的。

缺点：由于不确定要多少次才会出现重复，所以最终集合可能变得特别大，非int类型的情况以及使用递归时同理。

解法二：快慢指针法

思路：使用“快慢指针”思想找出循环：“快指针”每次走两步，“慢指针”每次走一步，当二者相等时，即为一个循环周期。此时，判断是不是因为1引起的循环，是的话就是快乐数，否则不是快乐数。

快慢指针

指针a移动速度快于指针b，当a的内容等于b的内容时，说明出现了循环，可以解决遍历过程需要保存的内容暴涨的情况。
